# Проект Image Processor

Этот проект представляет собой модульную систему обработки изображений на Rust, предназначенную для применения различных эффектов через динамически подключаемые плагины. Основной процессор загружает изображение, применяет плагин (например, размытие или зеркальное отражение) и сохраняет результат.

## Обзор

- **Основной процессор**: `image_processor` — читает изображение, загружает плагин, применяет его и сохраняет результат в формате PNG.
- **Плагины**:
  - `blur_plugin`: Применяет эффект размытия в зависимости от радиуса и количества итераций.
  - `mirror_plugin`: Выполняет горизонтальное, вертикальное или комбинированное зеркальное отражение изображения.

Плагины компилируются как динамические библиотеки (`.dll`, `.so`, `.dylib`) и взаимодействуют с процессором через FFI-интерфейс, совместимый с C.

## Интерфейс плагинов

Плагины должны экспортировать функцию со следующей сигнатурой:

```c
void process_image(
    unsigned long width,
    unsigned long height,
    unsigned char* rgba_data,
    const char* params_json
);
```

Параметр `params_json` — это строка, завершаемая нулём, содержащая параметры в формате JSON.

## Параметры плагинов

### Плагин размытия (Blur Plugin)

- `radius`: Радиус размытия в пикселях (по умолчанию: 1)
- `iterations`: Количество проходов размытия (по умолчанию: 1)

Пример:
```json
{"radius": 2, "iterations": 3}
```

### Плагин зеркального отражения (Mirror Plugin)

- `horizontal`: Если `true`, отражение по горизонтали (по умолчанию: false)
- `vertical`: Если `true`, отражение по вертикали (по умолчанию: false)

Пример:
```json
{"horizontal": true, "vertical": false}
```

## Использование

`image_processor` можно использовать как библиотеку или через CLI-интерфейс:

```bash
image_processor --input input.png --output output.png --shared-path plugins/libblur_plugin.so --params-path params.json
```

## Сборка

1. Сборка основного процессора:
   ```bash
   cd image_processor
   cargo build
   ```

2. Сборка плагинов:
   ```bash
   cd blur_plugin
   cargo build --release
   # Результат: target/release/libblur_plugin.so (или .dll)
   
   cd ../mirror_plugin
   cargo build --release
   # Результат: target/release/libmirror_plugin.so (или .dll)
   ```

3. Запуск с примером параметров:
   ```bash
   ./target/debug/image_processor --input example/in.png --output out.png --shared-path blur_plugin/target/release/libblur_plugin.dll --params-path example/blur.json.txt
   ```

## Примеры файлов параметров

- `example/blur.json.txt` — Размытие с радиусом 2 и 3 итерациями
- `example/horizontal.json.txt` — Горизонтальное отражение
- `example/vertical.json.txt` — Вертикальное отражение
- `example/turn.json.txt` — Комбинированное отражение (поворот на 180°)

## Архитектура

- Процессор загружает изображение в формате PNG.
- Параметры считываются из JSON-файла и передаются как C-строка.
- Плагин динамически загружается, и вызывается его функция `process_image`.
- Изменённое изображение сохраняется в формате PNG.

## Безопасность

- Вызовы FFI помечены как `unsafe`.
- Плагины должны гарантировать корректность указателей и размеров буферов.
- Невалидный JSON или некорректные параметры могут привести к неопределённому поведению.
